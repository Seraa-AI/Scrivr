Because Inscribe renders text manually using the HTML5 `<canvas>` API rather than the DOM, the layout engine heavily relies on accurate font metrics from the browser's `TextMeasurer`.

This creates a race condition known as the **"Canvas Font Trap."**

---

## The Canvas Font Trap

When you use the `<canvas>` API's `measureText()` method, the browser measures the text using the exact font family currently specified in the context context. However:

1. If you specify a custom web font (e.g., from Google Fonts or an `@font-face` declaration).
2. And that font is **not yet fully loaded** over the network.
3. The browser will temporarily fallback to a system font (e.g., Arial or Times New Roman).
4. `measureText()` will return the widths for the **fallback font**.

Because Inscribe's custom layout engine (`PageLayout.ts`) permanently absolute-positions every character and line break based on these initial measurements, your text layout will be completely corrupted once the custom font finally loads.

<Callout type="warn">
  Never initialize the editor or call `useCanvasEditor` until you can guarantee your application's fonts are fully loaded and ready in the browser.
</Callout>

---

## Safe Initialization with `document.fonts.ready`

The most robust way to sidestep this issue in a web environment is to wrap your editor in a React component that waits for the `document.fonts.ready` Promise to resolve.

### Example Wrapper Component

```tsx title="EditorWrapper.tsx"
import { useState, useEffect } from "react";
import { useCanvasEditor } from "@inscribe/react";
import { Editor } from "./Editor";

export function EditorWrapper() {
  const [fontsLoaded, setFontsLoaded] = useState(false);

  useEffect(() => {
    // Wait for all custom fonts declared in CSS/HTML to finish loading
    document.fonts.ready.then(() => {
      setFontsLoaded(true);
    });
  }, []);

  // Display a fallback UI while fonts are downloading
  if (!fontsLoaded) {
    return (
      <div className="flex items-center justify-center p-10">
        <p className="text-gray-500 font-mono">Loading typography...</p>
      </div>
    );
  }

  // Safe to mount — TextMeasurer will have 100% accurate metrics
  return <Editor />;
}
```

## Dynamic Font Loading (FontFace API)

If you are building an advanced editor that allows users to change document fonts on the fly (e.g., selecting "Merriweather" from a dropdown menu), you must explicitly load the font using the `FontFace` API and force the layout engine to recalculate everything.

```ts
async function changeDocumentFont(editor, fontFamily, fontUrl) {
  // 1. Fetch and register the new font
  const font = new FontFace(fontFamily, `url(${fontUrl})`);
  await font.load();
  document.fonts.add(font);

  // 2. Update your EditorState or PageConfig with the new font family name
  // ... (your state management here)

  // 3. Mark the editor as dirty and force a full re-measure
  editor.dirty = true;
  editor.ensureLayout();
}
```

<Callout type="info">
  When using `editor.ensureLayout()`, Inscribe re-runs `layoutDocument()` and `populateCharMap()`. This guarantees that the spatial index is updated with the new `measureText()` dimensions corresponding to your newly loaded font.
</Callout>
