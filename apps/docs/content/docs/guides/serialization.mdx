title: Serialization
description: Reading and writing document content — JSON, Markdown, and programmatic content loading.

Inscribe stores the document as an immutable ProseMirror `EditorState`. This guide covers how to
serialize it for storage and how to load content back into the editor.

---

## Reading Content

### JSON

The most reliable format — lossless round-trip with full schema fidelity:

```ts
const state = editor.getState();
const json = state.doc.toJSON();
// Save to your backend / localStorage:
localStorage.setItem('doc', JSON.stringify(json));
```

JSON output mirrors the ProseMirror node structure:

```json
{
  "type": "doc",
  "content": [
    {
      "type": "heading",
      "attrs": { "level": 1, "align": "left" },
      "content": [{ "type": "text", "text": "Hello" }]
    },
    {
      "type": "paragraph",
      "content": [{ "type": "text", "text": "World" }]
    }
  ]
}
```

### Markdown

```ts
const markdown = editor.getMarkdown();
// "# Hello\n\nWorld"
```

`getMarkdown()` uses extension-contributed serializer rules. Marks and nodes without rules
(e.g. `image`) are serialized as their closest Markdown equivalent or omitted.

## Loading Content

### At construction time

Pass the document JSON as `doc` in `EditorOptions`. The editor initialises its state from
it instead of starting with an empty document:

```ts
import { Editor, StarterKit } from '@inscribe/core';

const saved = await db.getDocument(id);

const editor = new Editor({
  extensions: [StarterKit],
  doc: saved.content, // ProseMirror JSON
});
```

### After mount with `setContent`

Use the `setContent` command to replace the document at any time:

```ts
const saved = JSON.parse(localStorage.getItem('doc') ?? 'null');
if (saved) {
  editor.commands.setContent(saved);
}
```

A good pattern is to load content in the `onCreate` callback:

```ts
const editor = useInscribeEditor({
  extensions: [StarterKit],
  onCreate: ({ editor }) => {
    const saved = localStorage.getItem('doc');
    if (saved) {
      editor.commands.setContent(JSON.parse(saved));
    }
  },
});
```

## Auto-Save on Change

Use `onUpdate` (React) or `onChange` (headless) to persist on every edit. Always debounce to
avoid excessive writes:

```ts
// React
const editor = useInscribeEditor({
  extensions: [StarterKit],
  onUpdate: ({ editor }) => {
    const json = editor.getState().doc.toJSON();
    debouncedSave(json);
  },
});

// Headless
const editor = new Editor({
  extensions: [StarterKit],
  onChange: (state) => {
    debouncedSave(state.doc.toJSON());
  },
});
```

## Markdown Import

To programmatically load Markdown, use the bundled parser:

```ts
import { markdownToDoc } from '@inscribe/core';

const doc = markdownToDoc(markdownString, editor.schema);
editor.commands.setContent(doc.toJSON());
```

## Clearing the Document

```ts
// Minimal valid document — a single empty paragraph
editor.commands.setContent({ type: 'doc', content: [{ type: 'paragraph' }] });
```

## Server-side Serialization

Use `ServerEditor` from `@inscribe/server` to parse and transform documents in Node.js
without mounting a canvas:

```ts
import { ServerEditor } from '@inscribe/server';
import { StarterKit } from '@inscribe/core';

const server = new ServerEditor({
  extensions: [StarterKit],
  doc: saved.content,
});

// Read
const markdown = server.getMarkdown();
const json = server.getState().doc.toJSON();

// Apply a transaction programmatically
const state = server.getState();
server._applyTransaction(state.tr.insertText('Prepended text.\n\n', 0));

// Save
await db.saveDocument(id, server.getState().doc.toJSON());
```

<Callout type="info">
  `ServerEditor` accepts the same `EditorOptions` as the browser `Editor` but never renders
  a canvas. It is safe to use in Next.js Route Handlers, Hono endpoints, or any Node.js
  environment.
</Callout>
