`StarterKit` is the batteries-included default for Inscribe. It bundles everything needed for a fully
functional rich-text document editor.

---

## How to Use StarterKit

```ts
import { StarterKit } from '@inscribe/core'; // or '@inscribe/react'

const editor = new Editor({ extensions: [StarterKit] });
```

To selectively disable or configure individual extensions:

```ts
StarterKit.configure({
  heading: { levels: [1, 2, 3] }, // limit to H1–H3
  history: false,                  // disable undo/redo
  image: false,                    // remove image support
})
```

## Block Nodes

| Extension | Node type | Description | Input rule |
|---|---|---|---|
| `Paragraph` | `paragraph` | Default text block | — |
| `Heading` | `heading` | H1–H6 with `level` attr | `# `, `## ` … |
| `List` | `bulletList`, `orderedList`, `listItem` | Nested lists | `- `, `1. ` |
| `CodeBlock` | `codeBlock` | Monospace fenced block | ` ``` ` |
| `HorizontalRule` | `horizontal_rule` | Divider line | `---` |
| `Image` | `image` | Inline image with `src`, `alt`, `width`, `height` | — |

## Utilities (Not Node Types)

| Extension | Description |
|---|---|
| `Alignment` | Text alignment via `align` attr on block nodes (`left`, `center`, `right`, `justify`) |
| `History` | Undo/redo via ProseMirror `history` plugin |

## History (Undo / Redo)

Undo and redo are provided by the `History` extension (included in `StarterKit`). No extra
configuration is needed — `Mod-Z` and `Mod-Shift-Z` work out of the box.

```ts
// Programmatic undo/redo
editor.commands.undo();
editor.commands.redo();
```

To configure history depth:

```ts
StarterKit.configure({
  history: {
    depth: 150,         // max undo steps (default: 100)
    newGroupDelay: 500, // ms before a new undo group is created
  }
})
```

To disable history (e.g. when using Yjs for collaborative undo):

```ts
StarterKit.configure({ history: false })
```
