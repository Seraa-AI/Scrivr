---
title: Event Callbacks
description: Full signature reference for all Editor and useInscribeEditor event callbacks.
---

Inscribe exposes event callbacks at two levels:

- **`EditorOptions`** — used when instantiating `Editor` directly (headless)
- **`UseInscribeEditorOptions`** — React hook wrapper with additional lifecycle callbacks

---

## EditorOptions Callbacks

These are passed directly to `new Editor(options)`.

### `onChange`
```ts
onChange?: (state: EditorState) => void
```
Fired synchronously on every state change — any document edit or selection move that produces a new
`EditorState`. The `state` argument is the new (immutable) state.

This is the ideal hook for serializing the document for persistence:

```ts
const editor = new Editor({
  extensions: [StarterKit],
  onChange: (state) => {
    const json = state.doc.toJSON();
    localStorage.setItem('doc', JSON.stringify(json));
  },
});
```

<Callout type="info">
  When using `<Inscribe />` and `useInscribeEditor`, you do not need `onChange` — use `onUpdate` instead.
  The React adapter subscribes to the editor internally via `editor.subscribe()`.
</Callout>

---

### `onFocusChange`
```ts
onFocusChange?: (focused: boolean) => void
```
Fired when the editor's hidden textarea gains or loses focus. `focused` is `true` on focus-in,
`false` on focus-out.

```ts
const editor = new Editor({
  extensions: [StarterKit],
  onFocusChange: (focused) => {
    toolbar.classList.toggle('visible', focused);
  },
});
```

---

### `onCursorTick`
```ts
onCursorTick?: (isVisible: boolean) => void
```
Fired on every cursor blink tick (every ~530ms) and immediately after any user interaction that
moves the cursor. `isVisible` alternates between `true` and `false` to drive the blink animation.

This is the hook for custom overlay renderers that need to redraw the cursor:

```ts
const editor = new Editor({
  extensions: [StarterKit],
  onCursorTick: (isVisible) => {
    overlayCanvas.drawCursor(isVisible);
  },
});
```

<Callout type="info">
  When using `<Inscribe />`, the `ViewManager` handles cursor repainting automatically.
  You only need `onCursorTick` when building a completely custom rendering adapter.
</Callout>

---

## UseInscribeEditorOptions Callbacks (React)

These are passed to `useInscribeEditor(options, deps?)`. They are a superset of `EditorOptions`
callbacks, using a `{ editor }` props object for consistency with the Tiptap API.

### `onCreate`
```ts
onCreate?: (props: { editor: Editor }) => void
```
Called once after the editor instance has been created. Use for one-time setup that requires access
to the editor instance (e.g. loading initial content):

```ts
const editor = useInscribeEditor({
  extensions: [StarterKit],
  onCreate: ({ editor }) => {
    editor.commands.setContent(savedDoc);
  },
});
```

---

### `onUpdate`
```ts
onUpdate?: (props: { editor: Editor }) => void
```
Called on every document or selection change (equivalent to `onChange` in `EditorOptions`).

```ts
const editor = useInscribeEditor({
  extensions: [StarterKit],
  onUpdate: ({ editor }) => {
    const md = editor.getMarkdown();
    setMarkdownPreview(md);
  },
});
```

---

### `onSelectionUpdate`
```ts
onSelectionUpdate?: (props: { editor: Editor }) => void
```
An alias for `onUpdate` provided for Tiptap API compatibility. Both fire on every state change.

---

### `onFocus`
```ts
onFocus?: (props: { editor: Editor }) => void
```
Fired when the editor textarea gains focus.

---

### `onBlur`
```ts
onBlur?: (props: { editor: Editor }) => void
```
Fired when the editor textarea loses focus.

---

### `onDestroy`
```ts
onDestroy?: () => void
```
Called just before the editor instance is destroyed (when the component unmounts or the dependency
array changes). Use to perform any external cleanup tied to this editor instance.

---

## Subscription vs Callbacks

For custom rendering adapters or framework-agnostic integrations, use `editor.subscribe()` instead
of callbacks:

```ts
const unsubscribe = editor.subscribe(() => {
  // fires on every state change, focus change, and cursor tick
  repaintOverlay();
});

// later
unsubscribe();
```

`editor.subscribe()` is the underlying primitive that both `useInscribeEditor` and `<Inscribe />`
use internally.
