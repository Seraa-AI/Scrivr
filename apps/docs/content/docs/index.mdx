## What is Inscribe?

**Inscribe** is an open-source document editor framework that renders entirely on `<canvas>` — not the DOM. It is purpose-built for applications that require **paginated, print-accurate documents**: think word processors, contract editors, or report builders.

> [!NOTE]
> Inscribe's layout engine is fully independent of the browser's CSS engine. Documents are visually **identical across all browsers, devices, and in exported PDFs** — with no font-rendering surprises.

If you've built with TipTap or used ProseMirror directly, Inscribe keeps the ProseMirror document model you already know, and replaces the DOM `EditorView` with a custom canvas renderer and a pagination-aware layout engine on top.

---

## Why Canvas?

Traditional DOM-based editors delegate all layout decisions to the browser — line-breaking, glyph placement, and pagination are all browser-controlled. This makes it nearly impossible to produce documents that are visually identical to their exported PDF counterparts.

Inscribe solves this by owning the entire render pipeline:

| Layer | Technology | Purpose |
| --- | --- | --- |
| **Model** | [ProseMirror](https://prosemirror.net/) | Manages the immutable document state, schema validation, and undo/redo history. |
| **Layout** | Custom Engine | Computes pagination, line‑breaking, and text measurement, independent of the DOM. |
| **Renderer** | HTML5 Canvas | Paints every glyph, cursor, and selection with pixel-perfect fidelity. |
| **Input** | Hidden `<textarea>` | Bridges keyboard and IME events safely to ProseMirror transactions. |

## Feature Highlights

- 📄 **Paginated layout** — fixed-size pages (A4 by default) with configurable margins and virtual scrolling
- 🎨 **Canvas rendering** — pixel-perfect output, zero dependency on browser CSS layout
- 🔄 **Real-time collaboration** — CRDT-based sync via Yjs + Hocuspocus, out of the box
- 🤖 **AI Toolkit** — streaming AI writing assistant via Anthropic / Vercel AI SDK
- 🔍 **Track Changes** — tracked insertions and deletions with accept/reject support
- 📤 **Export** — paginated PDFs (`pdf-lib`), DOCX (OpenXML), and Markdown
- 🧩 **Extension-first** — compose only the features you need; built-in extensions for bold, headings, lists, links, and more

## Key Concepts

**Extension-first architecture.** Every feature — headings, lists, links, collaboration — is an `Extension`. You compose your editor via `StarterKit` or an explicit extension list. Unused extensions don't ship.

**One source of truth.** The ProseMirror `EditorState` is the single document model. The canvas renderer reads it, Yjs syncs it across peers, and the export layer serializes it — nothing holds a shadow copy.

**Pages as first-class citizens.** Documents are broken into fixed-size pages. The `ViewManager` virtual-scrolls them, only painting what's in the viewport. Page dimensions and margins are controlled by a simple `PageConfig` object.

**`CharacterMap` as spatial index.** After each layout pass, every character has a known absolute `(x, y, page)` coordinate. This enables fast hit-testing — *"which character is under my cursor?"* — without any DOM measurements.
