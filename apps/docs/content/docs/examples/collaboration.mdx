This example shows how to set up a real-time collaborative editor using `@inscribe/plugins`
and an `@inscribe/server` Hocuspocus backend.

---

## How It Works

Inscribe's collaboration layer uses **Yjs** CRDTs to merge concurrent edits. The library owns
the WebSocket connection — you don't need to manage a provider on the client.

- `Collaboration.configure({ url, name })` — connects to your server and binds the document
- `CollaborationCursor.configure({ user })` — renders remote user cursors on the overlay canvas
- `startReady: false` — delays layout rendering until the server has synced the initial document

---

## Backend Setup

Install the server package:

```bash
pnpm add @inscribe/server
```

```ts title="server.ts"
import { createServer } from '@inscribe/server';

const server = createServer({
  port: 1234,
  // Optional: persist documents to disk or a database
  onLoadDocument: async ({ documentName }) => {
    return await db.load(documentName);
  },
  onStoreDocument: async ({ documentName, document }) => {
    await db.save(documentName, document);
  },
});

server.listen();
console.log('Collaboration server running on ws://localhost:1234');
```

## Frontend Setup

```tsx title="CollaborativeEditor.tsx"
import { useInscribeEditor, Inscribe, StarterKit } from '@inscribe/react';
import { Collaboration, CollaborationCursor } from '@inscribe/plugins';

interface Props {
  documentId: string;
  user: { name: string; color: string };
}

export function CollaborativeEditor({ documentId, user }: Props) {
  const editor = useInscribeEditor(
    {
      extensions: [
        StarterKit,
        Collaboration.configure({
          url: import.meta.env.VITE_WS_URL, // e.g. 'ws://localhost:1234'
          name: documentId,
        }),
        CollaborationCursor.configure({ user }),
      ],
      // Suppress layout until the server syncs the initial document
      startReady: false,
    },
    [documentId], // re-initialize when document changes
  );

  const loadingState = useEditorState({
    editor,
    selector: ({ editor }) => editor?.loadingState,
  });

  if (loadingState === 'syncing') {
    return <div>Connecting…</div>;
  }

  return <Inscribe editor={editor} style={{ height: '100vh' }} />;
}
```

**Environment variable required:** `VITE_WS_URL` — the WebSocket URL of your Hocuspocus server.

---

## `loadingState`

The editor exposes a `loadingState` property to track the connection lifecycle:

| Value | Meaning |
|-------|---------|
| `'syncing'` | WebSocket connected, waiting for initial document sync from server. |
| `'rendering'` | Document synced, layout engine running for the first time. |
| `'ready'` | Layout complete — document is fully visible. |

Use `loadingState` to show a loading spinner or skeleton UI while the document loads.

## Cursor Colours

Each connected user should have a distinct colour. A common pattern is to assign colours from a
palette based on user ID:

```ts
const CURSOR_COLORS = [
  '#f7b731', '#26de81', '#45aaf2', '#fd9644', '#a55eea',
];

function getCursorColor(userId: string): string {
  let hash = 0;
  for (const char of userId) hash = (hash * 31 + char.charCodeAt(0)) & 0xffffffff;
  return CURSOR_COLORS[Math.abs(hash) % CURSOR_COLORS.length];
}
```

```ts
CollaborationCursor.configure({
  user: {
    name: currentUser.name,
    color: getCursorColor(currentUser.id),
  },
})
```
